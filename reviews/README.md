# Reviews

This folder stores structured product reviews — CEO reviews, engineering reviews, and other formal assessments of the product plan, research, or implementation.

## Convention

One file per review. Use the naming pattern `YYYY-MM-review-type.md`. Add a row to the index below when filing a new review.

## Index

| File | Type | Date | Scope | Status |
|---|---|---|---|---|
| [2026-04-ceo-review.md](./2026-04-ceo-review.md) | CEO review | 2026-04-26 | Research folder review — post-pivot product plan | Complete — hand to eng-review after Step 3 |

## Notes

- Review files are read-only once filed.
- Actions arising from a review should be tracked as GitHub issues or referenced in `PLAN.md`.
